# Talk-Story

> *一个 Claude Code 技能，通过苏格拉底式对话——而非讲座——将一个模糊的研究想法发展成可发表的论文设计。*

[English](./README.md) | 中文

---

## 这是什么？

大多数研究想法起步于一个模糊的观察：

> *"我注意到 X 现象在中国似乎有些不同……"*  
> *"这个东西能发论文吗？"*  
> *"我有个直觉，但说不清楚。"*

**Talk-Story 是一个结构化的思维伙伴。** 它不会验证你的想法——它会磨砺它。通过七个递进阶段的苏格拉底式对话，它会反驳你、暴露薄弱环节，帮你发现这个想法是否真正站得住脚。

输出不是一份总结，而是一段 3–4 句话的陈述——你可以直接用它开头一篇顶级期刊投稿。

---

## 七个阶段

| # | 阶段 | 解决的问题 |
|---|------|------------|
| 1 | 澄清观察 | 你到底看到了什么？在哪个人群/情境中？ |
| 2 | 识别谜题 | 为什么这个发现令人惊讶？理论预测的是什么？ |
| 3 | 找到贡献 | 读者读完你的论文，会知道什么他之前不知道的事？ |
| 4 | 构建机制 | 这个关系*为什么*存在——一步一步地说清楚？ |
| 5 | 接入文献争论 | 这篇论文进入了哪些正在进行的学术争论？ |
| 6 | 评估可行性 | 你有数据、方法和时间吗？ |
| 7 | 电梯演讲 | 一段 3–4 句话的综合陈述，证明想法已经成熟 |

当你能流畅地呈现第七阶段，并能说出目标期刊，对话才算完成。

## 设计原则

**每次只走一步。** 每轮最多两个问题——研究者需要空间去思考，而不是被问题淹没。

**点名挑战类型。** 当反驳时，Talk-Story 始终会指明问题类型：*"这是一个贡献问题"* / *"这是一个机制问题"* / *"这是一个可行性问题"*。诊断精准，修正才能精准。

**不替你把话说完。** 当你快要想到某个洞察时，Talk-Story 会提问，而不是替你说出来。这个想法必须是你自己的——不是被喂给你的。

**双语。** 用你使用的语言回应——中文、英文或混合。语码转换没问题。

**入口，不是终点。** Talk-Story 设计为在想法成熟后，交棒给 `deep-research` 或 `academic-paper`。

## 使用方法

自然触发——不需要 slash 命令。只需告诉 Claude 你有一个研究想法：

```
"我有个关于 X 的想法，能做成论文吗？"
"我想写 Y，这个能发吗？"
"I have an idea about X, can this become a paper?"
"这个现象能发什么级别的期刊？"
```

Claude 会以这句话开场：

> 好，把你的想法告诉我——一句话就好。不用完整，不用好听，就是你脑子里现在最原始的那个观察。

然后对话开始。

---

## 对话示例

```
你：   农村老人的自杀率比城市高，这能做成论文吗？

Claude：这是个有实质感的观察。先告诉我：
        这个差距是你在数据里发现的，还是你读到的某个统计？
        现有研究对这个模式有什么解释？

你：   我在数据里看到的。文献主要说是社会支持减少。

Claude：社会支持的解释本身有点宽泛——这是一个机制问题。
        如果是支持减少，那城市老人的支持也在减少，为什么差距还在扩大？
        有没有可能你观察到的其实是两件不同的事情？
```

## 贡献等级

在第三阶段工作时，Talk-Story 使用这个等级来推动你寻找更强的贡献：

| 强度 | 类型 | 示例 |
|------|------|------|
| ★★★★ | 打破共识 | "所有人都以为 X；这篇论文证明了 non-X" |
| ★★★ | 解决争论 | "A 派说 X，B 派说 Y；这篇论文说明了各自成立的条件" |
| ★★ | 拓展发现 | "X→Y 在发达国家成立；本文证明它在中国成立/不成立" |
| ★ | 记录现象 | "没有人在这个情境中测量过 X" |

如果你的贡献是"我用了新数据"或"我研究的是中国"——预期会受到反驳：*"那是情境，不是贡献。"*

## 交棒

Talk-Story 之后的自然下一步：

- **`deep-research`** — 系统性文献检索，绘制你的论文进入的学术争论地图
- **`academic-paper`** — 逐章规划论文结构并开始写作
- **书面研究简报** — 可发送给合作者或导师的可分享摘要

---

## 适用人群

社会学、经济学、人口学、公共卫生及相关社会科学领域的**研究生和研究人员**，在投入一个完整研究项目之前需要一个严格的思维伙伴。

如果你曾经写了三稿引言，才意识到论文根本没有贡献——这个技能是为你准备的。
